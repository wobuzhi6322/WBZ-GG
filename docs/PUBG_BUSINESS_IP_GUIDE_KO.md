# WBZ 상용 서비스용 PUBG 데이터/IP 가이드

## 1. 목적
- WBZ에서 PUBG 관련 기능을 상용 운영할 때 저작권/상표/IP 리스크를 줄이는 실무 기준입니다.
- 아래 항목은 법률 자문을 대체하지 않으며, 최종 출시 전 법무 검토가 필요합니다.

## 2. 데이터 소스 원칙
- 전적/시즌/리더보드: `developer.pubg.com`의 공식 API만 사용합니다.
- 공지/패치/맵 로테이션: `pubg.com` 공식 뉴스/맵 서비스 리포트만 사용합니다.
- 스팀 프로필 정보: Steam Web API 정책 범위 내에서만 사용합니다.
- 제3자 사이트(예: 커뮤니티 DB)의 데이터/텍스트/레이아웃은 크롤링 복제하지 않습니다.

## 3. 저작권/상표 실무 규칙
- KRAFTON 로고, PUBG 로고, 공식 아트워크는 무단 편집/재배포하지 않습니다.
- 맵/아이템 이미지는 공식 배포 자산 또는 사용 허용 범위가 명확한 자료만 사용합니다.
- 타 서비스 UI를 그대로 복제하지 않고, 기능 아이디어만 참고하여 자체 UI로 구현합니다.
- 스킨/아이템 드랍 확률은 공식 공지 기준으로만 표기하고, 출처/기준일을 표시합니다.
- 페이지 하단 또는 정보 패널에 `출처: PUBG 공식` 문구 및 원문 링크를 표시합니다.

## 4. 금지 사항
- 공식 API 약관을 우회하는 데이터 수집(비공식 엔드포인트 대량 스크랩) 금지.
- PUBG 공식 문서/타 사이트 텍스트를 문단 단위로 복붙 금지.
- 타 서비스의 지도 오버레이, 아이콘 세트, 좌표 DB를 그대로 복제 금지.
- `PUBG`, `BATTLEGROUNDS` 상표를 자사 상표처럼 오인되게 사용 금지.

## 5. 운영 체크리스트
- 기능별 `데이터 출처`, `갱신 주기`, `캐시 TTL`, `최종 동기화 시간`을 기록합니다.
- API 키는 `.env`로 분리하고 로그/클라이언트에 노출하지 않습니다.
- 서버 캐시 + `stale-while-revalidate`로 공식 소스 호출량을 제한합니다.
- 장애 시 이전 캐시 데이터로 폴백하고, “동기화 지연” 배지를 노출합니다.
